# 0xSonata - Presentación Detallada

## ¿Qué es esta presentación?

Esta es una presentación interactiva de **0xSonata**, un proyecto que ayuda a artistas musicales que usan Inteligencia Artificial (IA) a proteger su trabajo.

**Formato**: Presentación web con 12 slides interactivos  
**Duración estimada**: 10-15 minutos  
**Público**: Artistas, productores musicales, jueces de hackathon, inversores

---

## Cómo Usar Esta Presentación

1. **Abrir el archivo**: Abre `index.html` en tu navegador (Chrome, Firefox, Edge)
2. **Navegar**: Usa las flechas del teclado (← →) o los botones en pantalla
3. **Interactuar**: Puedes arrastrar al robot "BIT-BOT" por la pantalla
4. **Idioma**: Cambia entre Español e Inglés con el toggle en la esquina superior

## 📍 Slide 3: Cómo Funciona 0xSonata (Los 5 Pasos)

### ¿Qué se muestra?

```

└─────────────────────────────────────────┘
```

### ¿Qué decir?

> "0xSonata documenta TODO tu proceso creativo. No es solo decir 'yo lo hice'. Es registrar CADA decisión que tomaste, desde el prompt inicial hasta el archivo final. Cada paso genera un hash con timestamp que nadie puede modificar."

### Los 5 Pasos Explicados (Con Ejemplo)

Imagina que **Carlos** quiere crear una canción de reggaeton:

#### **Paso 1: Prompt** 🤖
```
Lo que Carlos escribe en Suno:
"Reggaeton beat, 95 BPM, estilo Daddy Yankee, 
letra sobre superación personal, voz masculina"
```
**¿Por qué importa?** Muestra que Carlos dio una dirección creativa específica.

#### **Paso 2: Variaciones** ✍️
```
Suno genera 4 opciones:
- Variación 1: Muy lenta
- Variación 2: Muy rápida
- Variación 3: ¡Perfecta! (Carlos elige esta)
- Variación 4: Buena pero no tanto
```
**¿Por qué importa?** Demuestra que hubo múltiples opciones y Carlos tomó una decisión.

#### **Paso 3: Tu Selección** ⛓️
```
Carlos escribe POR QUÉ eligió la Variación 3:
"Elegí esta porque el groove de batería era 
más orgánico y el bajo tenía más punch"
```
**¿Por qué importa?** Esta es la **DECISIÓN HUMANA** que el Copyright Office busca.

#### **Paso 4: Edición en DAW** 📜
```
Carlos abre la canción en FL Studio y:
- Agrega su propia voz rapeando
- Añade una línea de bajo extra
- Corta el intro (muy largo)
- Agrega efectos de reverb
```
**¿Por qué importa?** Muestra aporte humano más allá de la IA.

#### **Paso 5: Master Final** 💾
```
Carlos exporta el archivo final:
"MiCancion_Master_Final.wav"
→ Se genera el hash SHA-256 final
→ Se registra en blockchain con timestamp
```
**¿Por qué importa?** Es la evidencia final de tu obra completa.

### ¿Qué es un Hash SHA-256? (Explicado Fácil)

```
Imagina que tienes una receta de cocina.

El hash es como convertir esa receta en 
un código único:

Receta: "2 huevos, 1 taza de harina..."
Hash:   "a3f5b8c9d2e1f4..."

Si cambias UN ingrediente (1 huevo en vez de 2):
Hash:   "z9y8x7w6v5u4t3..." ← ¡COMPLETAMENTE DIFERENTE!
```

**Características del hash:**
- ✅ Es único (como huella digital)
- ✅ No puedes revertirlo (no puedes obtener la receta del hash)
- ✅ Si cambias 1 bit, cambia TODO el hash
- ✅ Es verificable por cualquiera

### Posibles Preguntas

**P: ¿Tengo que completar los 5 pasos?**  
R: No es obligatorio, pero mientras más documentes, más sólida es tu evidencia. Mínimo recomienda los pasos 1, 3 y 5.

**P: ¿Puedo editar después?**  
R: No. Una vez registrado en blockchain, es inmutable. Pero puedes agregar pasos adicionales si te olvidaste algo.

**P: ¿Qué es un DAW?**  
R: Es un programa para editar música (como FL Studio, Ableton, Logic Pro). Es como un Photoshop pero para audio.

┌─────────────────────────────────────┐

# Glosario de Términos

| Término | Significado Simple |
|---------|-------------------|
| **Blockchain** | Libro de contabilidad digital que nadie puede modificar |
| **Smart Contract** | Programa que se ejecuta solo cuando se cumplen condiciones |
| **Hash** | Huella digital única de un archivo |
| **Timestamp** | Fecha y hora oficial registrada |
| **NFT** | Token que representa propiedad de algo único |
| **Wallet** | Billetera digital para criptomonedas |
| **Gas** | Comisión por usar blockchain |
| **Mainnet** | Red principal de producción (dinero real) |
| **Devnet** | Red de desarrollo (dinero falso, para testing) |
| **Stake** | Depósito de garantía |
| **Tier** | Nivel de reputación |
| **DAO** | Organización descentralizada (no aplica aquí aún) |
| **zkRollup** | Tecnología que hace blockchain más rápido y barato |
| **ERC-721** | Estándar para NFTs en Ethereum/Syscoin |
| **TypeORM** | Herramienta para conectar base de datos con código |
| **NestJS** | Framework para hacer APIs modernas |
| **Angular** | Framework para hacer interfaces web |
| **Pali Wallet** | Wallet oficial de Syscoin |
| **tSYS** | SYS de testing (no vale dinero real) |
